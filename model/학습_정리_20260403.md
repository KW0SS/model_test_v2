## 3. 데이터/라벨 규칙 변경 이력

### 3-1. 초기 데이터

- 파일: `재무비율_20260402_182020.csv`

### 3-2. 상폐 전년도 추가 데이터

- 파일: `재무비율_20260402_205612.csv`
- 배경:
  - 상폐기업 데이터에 상폐 전년도까지 추가 반영
- 결과:
  - 양성 행 수는 증가했지만, 같은 회사의 여러 연도가 섞이면서 문제 난이도가 높아짐

### 3-3. 상폐 1년 전만 양성, 2년 전/3년 전 음성 규칙

파이프라인 수정 후 `--use-last-year` 옵션 의미를 아래처럼 변경했다.

- 상폐 1년 전: `label=1`
- 상폐 2년 전: `label=0`
- 상폐 3년 전: `label=0`

이 규칙으로 생성된 파일:

- `재무비율_20260402_222920.csv`

### 3-4. 모델학습용 CSV 구조 도입

- 파일: `모델학습용_20260402_232248.csv`
- 특징:
  - 원본 30개 피처 외에 비율형 절대값 대체 컬럼 사용
  - `_missing` indicator 컬럼 포함
  - 총 feature 수: 60개

### 3-5. 전처리완료 CSV 도입

- 파일: `모델학습전처리완료_20260403_001717.csv`
- 추가 적용된 전처리:
  - feature 30개 결측치 중앙값 대체
  - 전부 결측인 컬럼은 0으로 대체
  - signed `log1p`
  - `RobustScaler`
- `_missing` indicator 30개도 함께 포함
- 총 feature 수: 60개


## 4. 모델 스크립트 변경 이력

### 4-1. `model_20260402_201545.py` 생성

주요 내용:

- 30개 재무비율 컬럼 사용
- `train/valid/test = 70/15/15`
- `LogisticRegression`, `RandomForest`, `XGBoost`
- validation `f1` 기준 최고 모델 저장

### 4-2. `model_20260402_213421.py` 생성

주요 내용:

- validation에서 threshold를 탐색해 `f1` 기준 최적 threshold 선정
- 그 threshold를 test에 그대로 적용

### 4-3. `model_20260402_213421.py` 구조 확장

추가 수정 내용:

- 메타 컬럼 5개와 `label`을 제외한 전체 feature를 자동 인식
- `모델학습용_20260402_232248.csv`처럼 60개 feature 구조도 학습 가능하도록 수정

### 4-4. 전처리완료 CSV 대응

추가 수정 내용:

- `_missing` 컬럼이 존재하는 경우 전처리완료 입력으로 간주
- 이미 전처리된 입력에 대해 추가 전처리를 다시 하지 않도록 수정
- 내부 fallback 로직:
  - 전처리완료 CSV: 그대로 사용
  - 원본형 CSV: 내부 전처리 적용 가능하도록 구조 유지
